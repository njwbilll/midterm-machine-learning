## Purpose

This repository contains the submission for the Machine Learning midterm assignment. The project demonstrates a complete end-to-end regression pipeline built to predict the release year of a song based on its audio characteristics. The goal is to apply and compare multiple machine learning approaches, from a simple linear baseline to a tuned gradient boosting model, while integrating tools for experiment tracking and model interpretability.

---

## Project Overview

The dataset consists of 90 numerical audio features — derived from timbre and signal properties — with the release year as the prediction target. The pipeline covers every stage from raw data loading through final model persistence, following the structure required for the assignment.

The core question the project explores is how well audio signal features can predict when a song was recorded. This is a non-trivial regression problem because musical styles evolve gradually and the same acoustic patterns can appear across multiple decades.

## Models

**Ridge Regression**
A linear regression model with L2 regularization (alpha=1.0). Used as the baseline to establish a performance floor. Ridge is appropriate here because it handles correlated features gracefully through regularization, though its expressiveness is limited to linear relationships in the feature space.

**Random Forest Regressor**
An ensemble of 100 decision trees trained with bagging (max_depth=12, min_samples_leaf=5). Random Forest can capture non-linear patterns and provides built-in feature importance, making it useful both as a model and as a diagnostic tool for understanding which audio features carry the most predictive signal.

**XGBoost with Optuna Tuning**
A gradient boosting model where trees are built sequentially, with each new tree correcting the residuals of the previous ensemble. Hyperparameters were searched using Optuna's Tree-structured Parzen Estimator (TPE) over 15 trials, optimizing for validation RMSE. Parameters tuned include n_estimators, max_depth, learning_rate, subsample, colsample_bytree, and min_child_weight.

## Evaluation Metrics

| Metric | Description |
|--------|-------------|
| MSE | Mean Squared Error — average squared deviation, sensitive to large errors |
| RMSE | Root Mean Squared Error — same unit as target (years), directly interpretable as average prediction offset |
| MAE | Mean Absolute Error — average absolute deviation, more robust to outliers than RMSE |
| R2 | Coefficient of determination — proportion of variance in the target explained by the model (higher is better, max 1.0) |

All metrics are computed on the held-out validation set (20% of the data).
